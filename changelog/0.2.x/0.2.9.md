# 0.2.9 — 2026-06-12

## Added

- **`libofcongress_search_subjects` truncation disclosure** — enrichment now carries `truncated`, `shown`, and `cap`; when results hit the requested `limit` the handler emits `ctx.enrich.truncated(...)` with guidance to raise the limit or refine the query.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into the `bundle` script: runs `mcpb clean`, then strips dependency-shipped agent-doc entries (`node_modules/**/skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach, and asserts none remain.
- **`scripts/lint-packaging.ts` bundle-content and identity guards** — checks that `.mcpbignore` anchors known dev-dir patterns (so an unanchored `skills/` can't strip runtime `node_modules/.../skills/`), that critical runtime paths survive, that a built `.mcpb` ships zero `node_modules/**` agent-doc entries, and that `createApp()` `name`/`title` and manifest `display_name` equal the unscoped package name.
- **`coerce-boolean-env-flag` antipattern rule** — `scripts/check-framework-antipatterns.ts` now flags `z.coerce.boolean()` on env flags (which can't be disabled via the environment) in favor of `z.stringbool()`.
- **Docker `HEALTHCHECK`** — container reports health by polling `/healthz`; `ARG APP_VERSION` feeds an `org.opencontainers.image.version` OCI label.

## Changed

- **`@cyanheads/mcp-ts-core` `^0.9.21` → `^0.10.6`** — framework adoption.
- **Display identity** — `createApp()` now sets `name` and `title` to `libofcongress-mcp-server` explicitly; `.codex-plugin/plugin.json` `interface.displayName` changed from `@cyanheads/libofcongress-mcp-server` to the unscoped `libofcongress-mcp-server`.
- **`.mcpbignore` patterns anchored** — dev-dir entries now use leading-slash anchors and add `/.agents/` and `/skills/`, scoping exclusions to the repo root.
