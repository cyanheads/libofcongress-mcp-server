# 0.2.14 — 2026-07-16

## Added

- **`libofcongress_get_item` returns eight more metadata fields** — `summary`, `call_number`, `languages`, `locations`, `former_ids`, `original_formats`, `online_formats`, and `access_restricted`, all normalized from the same upstream response with no extra request. Fields absent upstream are omitted; `access_restricted` is `typeof`-guarded so a genuine `false` survives, and `call_number` (retyped `string | string[]`) collapses via `extractFirstString`. ([#36](https://github.com/cyanheads/libofcongress-mcp-server/issues/36))
- **Typed `errors[]` contract on the `libofcongress://item` resource** — declares `item_not_found` (NotFound) and `rate_limit_exceeded` (RateLimited), remapped from the service's `McpError` codes via `ctx.fail`.

## Fixed

- **`libofcongress://item/{+item_id}` reads multi-segment IDs** — the template now uses RFC 6570 reserved expansion, which the SDK compiles to `(.+)` instead of `([^/,]+)`, so slash-separated newspaper IDs (`sn95047246/1935-09-05/ed-1`) match; the handler `decodeURIComponent`s the captured value once, so percent-encoded slashes resolve too. ([#34](https://github.com/cyanheads/libofcongress-mcp-server/issues/34))
- **`libofcongress_get_item` `format()` renders `resource_links` and `related_items` in full** — dropped the five-entry cap and `… and N more` overflow lines, so a `content[]`-only client reaches every value `structuredContent` holds. A dense item renders a long `content[]` block by design. ([#7](https://github.com/cyanheads/libofcongress-mcp-server/issues/7))
- **Internal request `url` no longer attached to error data** — `fetchJson` and `fetchSearchJson` drop `{ url }` from their `notFound`/`serviceUnavailable` throws, keeping the LOC endpoint and query string — which on the search path echoes the caller's own search terms — out of payloads that reach the client. Callers already hold the caller-facing identifier worth reporting. ([#1](https://github.com/cyanheads/libofcongress-mcp-server/issues/1))
