# 0.2.6 — 2026-05-30

## Changed

- **`libofcongress_search`, `libofcongress_search_newspapers`, `libofcongress_browse_collections`, `libofcongress_search_subjects`** — enrichment adoption: query echoes (`effectiveQuery`), true result totals (`totalCount`), and empty-result guidance (`notice`) now surface in a typed `enrichment` block that reaches both `structuredContent` and the `content[]` markdown trailer automatically.
- **`structuredContent`** — the `message` key (previously in the domain `output` block) has moved to `enrichment.notice`; value and presence semantics are preserved, both channels carry the guidance.
- **`libofcongress_search`, `libofcongress_search_newspapers`, `libofcongress_browse_collections`, `libofcongress_search_subjects`** — removed the unreachable `empty_results` error contract from all four tools ([#21](https://github.com/cyanheads/libofcongress-mcp-server/issues/21)); empty results return a success response with `enrichment.notice` guidance instead of a JSON-RPC error.

## Dependency bumps

- `@cyanheads/mcp-ts-core` ^0.9.13 → ^0.9.16
