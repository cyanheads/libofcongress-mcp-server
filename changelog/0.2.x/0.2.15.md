# 0.2.15 — 2026-07-16

## Added

- **Transient-fault resilience for LOC upstreams** — new `src/services/http.ts` (`timedFetch`, `isTransientNetworkFault`) adds a 30s per-request timeout ceiling composed with `ctx.signal`, plus a retry scoped to raw network faults and timeouts only. Every status-derived error — 429, `ServiceUnavailable`, the HTML soft-block, 404 — fails fast, so a retry can never re-hit LOC's ~1-hour IP block. Wraps `fetchJson`/`fetchSearchJson`, the newspaper OCR fetch, and the subject-suggest fetch. ([#30](https://github.com/cyanheads/libofcongress-mcp-server/issues/30))

## Fixed

- **`libofcongress_search_subjects` no longer reports false empties** — the id.loc.gov suggest request size is decoupled from `limit` and always draws the endpoint's full 50-candidate pool (new `SUGGEST_MAX_COUNT`), so an LCSH heading ranked below name-authority records is no longer sliced off by a small `limit`. An existing `ctx.enrich.notice` now distinguishes "candidate pool exhausted by ranking" (retry with a more specific query) from "no LCSH coverage", and truncation fires on `matchCount > limit`. No new tool or parameter. ([#25](https://github.com/cyanheads/libofcongress-mcp-server/issues/25))
- **Internal request `url` withheld from error data on the subject-suggest path** — `LcLinkedDataService.searchSubjects`'s `serviceUnavailable` throws drop `{ url }`, whose query string echoes the caller's own search terms, extending the no-internal-URL invariant established in 0.2.14 ([#1](https://github.com/cyanheads/libofcongress-mcp-server/issues/1)) to `id.loc.gov`.
- **Mocked test suite no longer pays live request pacing** — `LOC_REQUEST_DELAY_MS=0` moves to Vitest `env` in `vitest.config.ts`, landing before any module seeds `getServerConfig()`'s module-level `??=` cache; the old per-test `process.env` assignments ran after the seed and were dead. The suite runs in ~2.8s (was ~137s), guarded by `tests/config/server-config.test.ts`. ([#38](https://github.com/cyanheads/libofcongress-mcp-server/issues/38))
