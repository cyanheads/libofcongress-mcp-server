# 0.2.12 — 2026-07-13

## Added

- **`libofcongress_get_newspaper_page` derives `date` and `sequence` from the page URL** when the `?fo=json&at=resource` endpoint omits them — date from the LCCN path segment, sequence from the `sp` query parameter. Upstream values still win when present. ([#28](https://github.com/cyanheads/libofcongress-mcp-server/issues/28))

## Fixed

- **`libofcongress_search_subjects` filters suggest results to the LCSH subjects namespace**, dropping name-authority and childrensSubjects records that broke the downstream `subject` filter; over-fetches (bounded at 50) so filtering doesn't undershoot `limit`. ([#25](https://github.com/cyanheads/libofcongress-mcp-server/issues/25))
- **`libofcongress_browse_collections`, `libofcongress_search`, and `libofcongress_search_newspapers` re-enrich `totalCount` to 0 on empty results** — LOC reports a nonzero `pagination.total` for some no-match queries, which previously left `totalCount` out of sync with the returned `total`. ([#27](https://github.com/cyanheads/libofcongress-mcp-server/issues/27))
