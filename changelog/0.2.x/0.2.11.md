# 0.2.11 — 2026-07-13

## Added

- **`is_item` field on `libofcongress_search` results** — `true` for catalog items resolvable via `libofcongress_get_item`; `false` for non-item resources (collection landing pages, exhibit/research-guide pages, newspaper `/resource/` pages), which have no item record. ([#24](https://github.com/cyanheads/libofcongress-mcp-server/issues/24))
- **`bunfig.toml` supply-chain guard** — `minimumReleaseAge` blocks installing package versions published in the last 3 days (excludes `@cyanheads/mcp-ts-core`, adopted same-day); a Socket scanner (`@socketsecurity/bun-security-scanner`) scans every package before install. `devcheck.config.json`'s `depcheck.ignores` allowlists the scanner package since it's referenced only from `bunfig.toml`, not imported in source.
- **`.github/SECURITY.md`** — vulnerability-reporting policy.
- **`.gitattributes`** — normalizes line endings to LF, marks `bun.lock`/`CHANGELOG.md`/`docs/tree.md` as generated and binary assets as binary for PR diffs.

## Changed

- **`@cyanheads/mcp-ts-core` `^0.10.9` → `^0.10.14`** — framework refresh.
- **Tool descriptions** for `libofcongress_search` and `libofcongress_get_item` document the `is_item` contract and multi-segment id support.
- **Docker base image** pinned `oven/bun:1.3`/`1.3-slim` → `1.3.14`/`1.3.14-slim`; build-stage `bun install` now runs with `--ignore-scripts`.
- **`.github/FUNDING.yml`** trimmed to the two active platforms (`github`, `buy_me_a_coffee`).
- **Vendored skills and devcheck scripts re-synced to the framework** — 12 `skills/*/SKILL.md`, 4 `skills/*/references/*.md`, and 4 `scripts/*.ts` picked up upstream body drift; `devcheck.ts`'s outdated-package check now recognizes versions held back by `minimumReleaseAge` instead of flagging them as unexpected.

## Fixed

- **`libofcongress_search` id extraction preserves the full `/item/` path** — previously truncated multi-segment newspaper ids (e.g. `sn95047246/1935-09-05/ed-1`) to their last path segment. ([#24](https://github.com/cyanheads/libofcongress-mcp-server/issues/24))
- **`libofcongress_get_item` encodes multi-segment item ids per path segment** instead of the whole id, so internal slashes no longer flatten to `%2F` and deep newspaper-edition paths resolve. ([#24](https://github.com/cyanheads/libofcongress-mcp-server/issues/24))
- **`libofcongress_get_item` and the `libofcongress://item/{item_id}` resource normalize protocol-relative item urls** (`//lccn.loc.gov/...`) to absolute `https://`. ([#26](https://github.com/cyanheads/libofcongress-mcp-server/issues/26))
