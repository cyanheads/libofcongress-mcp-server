# 0.2.13 — 2026-07-15

## Added

- **`libofcongress_search` accepts `collection_slug`** — scopes a search to one curated collection through LOC's `/collections/{slug}/` endpoint, which takes the same query string and returns the same envelope, so existing filters and pagination carry over. Pass a slug from `libofcongress_browse_collections`; `enrichment.effectiveCollectionSlug` echoes the applied scope. ([#35](https://github.com/cyanheads/libofcongress-mcp-server/issues/35))
- **`incompatible_filters` (ValidationError)** rejects `format` + `collection_slug` before any request — each selects a different LOC base path, so no single request can honor both.
- **`collection_not_found` (NotFound)** names an unrecognized `collection_slug` and points recovery at `libofcongress_browse_collections`, keeping the service's internal request URL out of the failure.

## Fixed

- **`libofcongress_browse_collections` returns the routable slug** — extraction takes the first path segment after `/collections/` rather than anchoring to the end of the URL, which missed the `/about-this-collection/` subpath LOC returns on every browse result and fell through to a title-derived slug (`aaron-copland-collection` instead of the real `aaron-copland`). ([#37](https://github.com/cyanheads/libofcongress-mcp-server/issues/37))
- **`libofcongress_browse_collections` populates `item_count`** from the upstream top-level `count`, `typeof`-guarded so a genuine `count: 0` survives. Distinct from the nested `item.total`/`item.digitized`, which carry a different figure for the same result. ([#32](https://github.com/cyanheads/libofcongress-mcp-server/issues/32))
- **`normalizePagination` falls back to `results` only when it is numeric** — LOC sends a display range (`"1 - 3"`) under that numeric-sounding key, which previously stood in for a missing `total` and put a string where the pagination contract declares a number.
- **README and `docs/design.md` describe `collection_slug`** for collection scoping in place of a `partof` facet (`fa=partof:{slug}`), which returns zero results; a design-decision entry records why the facet approach does not work.
