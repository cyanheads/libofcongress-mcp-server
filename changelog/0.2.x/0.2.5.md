# 0.2.5 — 2026-05-28

## Changed

- **`landing.requireAuth: false`** set explicitly in `createApp()` — the hosted server is a public catalog; 0.9.13 changed the framework default to require auth when `MCP_AUTH_MODE` is `jwt`/`oauth`, so this preserves the prior behavior for unauthenticated visitors.
- **`@cyanheads/mcp-ts-core`** ^0.9.9 → ^0.9.13 — picks up:
  - **`MCP_HTTP_MAX_BODY_BYTES`** — configurable inbound request-body cap (default 1 MiB); oversized requests rejected with 413 before body parsing.
  - **HTTP session-init gate** — stateful HTTP mode rejects non-`initialize` requests without `Mcp-Session-Id` with 400.
  - **`httpErrorHandler` log-level split** — expected client errors (401, 403, 400, 404) now use `warning` instead of full error pipeline, reducing noise in logs.
  - **`GET /mcp` keywords** — `package.json` `keywords` now surfaced on the status JSON endpoint.
  - Plugin metadata scaffold templates (`.claude-plugin/`, `.codex-plugin/`).
  - `code-simplifier` external skill added.
  - `skills/migrate-mcp-ts-template/` removed.

## Added

- **`.env.example`** — `MCP_HTTP_MAX_BODY_BYTES` documented.
- **`manifest.json`** — `repository`, `homepage`, and `license` fields added.
- **`.claude-plugin/`** and **`.codex-plugin/`** — plugin metadata scaffolded.
- **`skills/code-simplifier/`** — synced from framework.

## Security

- **HTTP request-body limit** (`MCP_HTTP_MAX_BODY_BYTES`) — oversized payloads now rejected before the MCP SDK allocates a per-request `McpServer`, limiting the memory amplification surface for malformed large requests.
