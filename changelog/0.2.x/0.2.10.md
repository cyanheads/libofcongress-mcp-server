# 0.2.10 — 2026-06-20

## Added

- **`check-dependency-specifiers` devcheck step** — vendored `scripts/check-dependency-specifiers.ts` (framework [cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246)) hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json` dependency sections and `bun.lock` `workspaces`, catching a `bun update --latest` that writes a `latest` dist-tag into the lock and lets a later install re-resolve past an intentional hold.
- **Plugin marketplace manifest checks in `lint:packaging`** — `scripts/lint-packaging.ts` now validates `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` (framework [cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240)): non-empty descriptions, unscoped display identity, and a full-package-name `npx -y` install arg. Enabled via the new `devcheck.config.json` `packaging.pluginManifests` flag.

## Changed

- **`@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.9`** — framework refresh. Adds `ctx.content` for non-text content blocks ([cyanheads/mcp-ts-core#239](https://github.com/cyanheads/mcp-ts-core/issues/239)), classifies SELECT-shaped Canvas SQL prepare failures as `invalid_sql` with binder detail ([cyanheads/mcp-ts-core#236](https://github.com/cyanheads/mcp-ts-core/issues/236)), and fixes a `DuckdbProvider.describe()` ambiguous-column error on filtered table lists ([cyanheads/mcp-ts-core#235](https://github.com/cyanheads/mcp-ts-core/issues/235)).
- **Vendored skills + devcheck scripts re-synced to the framework** — 13 `skills/*/SKILL.md` and 6 `scripts/*.ts` picked up upstream 0.10.7 body drift and the fresh-scaffold devcheck guards ([cyanheads/mcp-ts-core#237](https://github.com/cyanheads/mcp-ts-core/issues/237), [cyanheads/mcp-ts-core#238](https://github.com/cyanheads/mcp-ts-core/issues/238), [cyanheads/mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [cyanheads/mcp-ts-core#243](https://github.com/cyanheads/mcp-ts-core/issues/243)).
