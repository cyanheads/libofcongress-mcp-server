# 0.2.16 — 2026-07-16

## Fixed

- **Pagination stops advertising pages past LOC's ~100,000-item retrieval ceiling** — LOC 302→400s on pages beyond ~100k items, so `normalizePagination()` now caps `pages`/`has_next` at that depth and `has_next` never promises an unretrievable page. A new `ceilingReached` signal drives a `notice` on both `structuredContent` and `content[]` disclosing the cap with partition recovery (date/subject/location; date/state for `libofcongress_search_newspapers`). ([#33](https://github.com/cyanheads/libofcongress-mcp-server/issues/33))
- **`total` reports matching items, not the page count** — `normalizePagination()` reads the total from LOC's `pagination.of`; LOC reports the item count there and the *page* count in the misleadingly-named `pagination.total`, which the `total` output field previously surfaced. Envelopes without `of` (legacy/mocked shapes) keep the prior reading. ([#33](https://github.com/cyanheads/libofcongress-mcp-server/issues/33))
- **Valid deep-page results no longer discarded** — the `pages > 0 && page > pages` "contradictory pagination" guard is removed from `libofcongress_search`, `libofcongress_search_newspapers`, and `libofcongress_browse_collections`; it discarded genuinely-served results when LOC's page count under-reported the retrievable depth (LOC returns HTTP 200 with real items past the computed last page). `normalizePagination()` now advances `pages` to a served page so `page <= pages` stays consistent. ([#33](https://github.com/cyanheads/libofcongress-mcp-server/issues/33))
- **`libofcongress_search` renders `is_item` for both states** — `format()` now emits a line for `is_item: true` (pass the ID to `libofcongress_get_item`) as well as the existing `false` line, so a `content[]`-only client gets the discriminator that was previously structured-only. ([#31](https://github.com/cyanheads/libofcongress-mcp-server/issues/31))
- **`libofcongress_get_newspaper_page` discloses OCR-retrieval misses on both surfaces** — a new `enrichment` block fires a `notice` when `ocr_available` is `true` but `ocr_text` is empty (a transient retrieval miss, distinct from a genuinely image-only page); the `format()` prose that carried this fact only into `content[]` is dropped to avoid double-rendering. ([#31](https://github.com/cyanheads/libofcongress-mcp-server/issues/31))
