# 0.2.2 ‚Äî 2026-05-24

## Fixed

- **`rights` crash** ([#13](https://github.com/cyanheads/libofcongress-mcp-server/issues/13)) ‚Äî `rights` and `rights_information` typed as `string | string[]`; joined to a single string before return, preventing a type crash when LOC returns an array.
- **OCR text extraction** ([#14](https://github.com/cyanheads/libofcongress-mcp-server/issues/14)) ‚Äî `tile.loc.gov` returns JSON (`{ "<key>": { full_text: "..." } }`), not ALTO XML; extraction rewritten to parse the JSON shape. OCR test suite updated to match.
- **Out-of-range pagination** ([#15](https://github.com/cyanheads/libofcongress-mcp-server/issues/15)) ‚Äî `libofcongress_search`, `libofcongress_search_newspapers`, and `libofcongress_browse_collections` now return a clear `message` when `page` exceeds the available range, using `pages === 0` as the LOC 400 sentinel.
- **Protocol-relative URL normalization** ([#16](https://github.com/cyanheads/libofcongress-mcp-server/issues/16)) ‚Äî LOC sometimes returns `//lccn.loc.gov/...` URLs; normalized to `https:` before surfacing.
- **`libofcongress_get_newspaper_page` error contract** ([#17](https://github.com/cyanheads/libofcongress-mcp-server/issues/17)) ‚Äî handler now `async`; `NotFound` errors from the service are re-thrown as typed `page_not_found` failures via `ctx.fail`. Redundant URL-parse validation removed (prefix check is sufficient).
- **`libofcongress_get_item` error contract** ([#18](https://github.com/cyanheads/libofcongress-mcp-server/issues/18)) ‚Äî handler converted to `async`; `NotFound` errors re-thrown as typed `item_not_found` failures via `ctx.fail`.
- **`libofcongress_search` and `libofcongress_search_newspapers` query input** ([#19](https://github.com/cyanheads/libofcongress-mcp-server/issues/19)) ‚Äî `query` field gains `.trim()` to strip accidental leading/trailing whitespace before hitting the LOC API.

## Changed

- **`libofcongress_browse_collections` description** ‚Äî removed incorrect claim that the `slug` value can be passed as a `partof` facet to `libofcongress_search` (that routing does not work); replaced with accurate description of the returned URL.
- **`slug` output field description** ‚Äî simplified to reflect actual semantics (identifies the collection in loc.gov URLs).
- **`libofcongress://item` resource description** ‚Äî removed redundant "Stable URI for injecting item context into agent conversations" sentence.
- **`libofcongress_get_newspaper_page` `page_url` description** ‚Äî replaced vague format description with a concrete example URL.
- **`HTML_RESPONSE_RE` regex** ‚Äî extracted to a module-level constant to avoid recompilation on every request.
- **`normalizePagination` stub replaced** ‚Äî `fetchSearchJson` returning `null` (LOC 400) now returns an explicit `{ pages: 0 }` sentinel object instead of calling the normalization helper with `undefined`.
