# 0.2.8 — 2026-06-04

## Fixed

- **`rate_limit_exceeded` error contract** — all 5 tool handlers now route rate-limit throws from the service layer through `ctx.fail('rate_limit_exceeded', ...)` so the framework's contract-correlation machinery auto-populates `retryable: false` and `recovery.hint`. Previously the service threw `rateLimited()` factory errors directly, bypassing the contract entry entirely ([#22](https://github.com/cyanheads/libofcongress-mcp-server/issues/22)).
- **`rate_limit_exceeded` error code** — corrected from `JsonRpcErrorCode.ServiceUnavailable` (-32000) to `JsonRpcErrorCode.RateLimited` (-32003) across all 5 tool contracts, matching the factory code thrown by the service layer.
- **`retryable: false`** — added to each `rate_limit_exceeded` contract entry; the framework's retry machinery now receives an explicit do-not-retry signal for the 1-hour block window.
- **Test fix** — `libofcongress-search.tool.test.ts` rate-limit test now passes `errors: locSearch.errors` to `createMockContext`, required for `ctx.fail` contract lookup.
